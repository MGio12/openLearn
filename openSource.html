Comprendre

L'open source, c'est quoi ?

Un logiciel open source, c'est un logiciel dont le code source est public. N'importe qui peut le lire, le comprendre, le modifier et l'améliorer. C'est l'opposé d'un logiciel propriétaire, où le code est un secret gardé par une entreprise.

L'open source repose sur une licence qui fixe les règles du jeu. La licence MIT, celle d'ObjectifLycée, est la plus permissive qui soit : tu peux faire ce que tu veux avec le code, tant que tu cites la source originale.

Lisible
Tout le monde peut lire le code et comprendre comment l'app fonctionne. Aucune boîte noire.
Forkable
Tu peux copier le projet, l'adapter, le faire évoluer dans une direction différente. C'est légal et encouragé.
Contributif
N'importe qui peut proposer une correction, une amélioration ou une nouvelle fonctionnalité.

Projets open source connus : Linux (le coeur de la plupart des serveurs du monde), Firefox, VS Code, Wikipedia, Python, React. Ces outils font tourner l'internet.

Savoir-faire

Les bonnes pratiques

L'open source a son vocabulaire, ses rituels, ses conventions. Voici les termes que tu entendras partout et ce qu'ils veulent vraiment dire.

Fork

Copier un projet dans ton propre espace GitHub pour expérimenter librement, sans risquer d'abîmer l'original. Point de départ de toute contribution.

Issue

Signaler un bug, poser une question ou proposer une idée directement sur le projet. Une bonne issue décrit le problème clairement, avec un exemple reproductible.

Pull request

Proposer une modification au projet original. Le mainteneur relira ton code, discutera avec toi, et fusionnera (ou non) ta contribution. C'est une conversation sur le code.

Code review

Relire le code d'un autre avant de le fusionner. On vérifie la logique, la lisibilité, les cas limites. C'est une compétence qui s'apprend et qui rend meilleur.

Licence MIT

Les règles du jeu : tu peux utiliser, modifier et redistribuer le code, même commercialement, tant que tu cites la source. La plus ouverte des licences courantes.

Documentation

Un bon projet s'explique lui-même. README clair, commentaires utiles, guide de contribution. La doc est souvent la meilleure première contribution.

Commit

Un enregistrement d'une modification dans l'historique Git. Un bon commit message dit pourquoi, pas juste quoi. L'historique Git est une mémoire collective.

Compétence clé

L'IA comme partenaire de code

ObjectifLycée a été construit avec Claude Code comme co-pilote. Pas pour remplacer la réflexion, mais pour accélérer la boucle hypothèse → test → correction. C'est une méthode, pas un raccourci.

Ce que l'IA fait bien
Expliquer une erreur en une phrase. Suggérer une refacto. Générer un squelette de code. Trouver un bug dans 50 lignes. Proposer des noms. Traduire une idée en code.
Ce que tu dois garder
Comprendre pourquoi ça marche. Juger si la suggestion est juste. Décider de l'architecture. Vérifier les effets de bord. Relire avant de fusionner. L'IA se trompe.

Savoir poser les bonnes questions à une IA est une compétence aussi importante que savoir coder. Ce projet est la preuve qu'un lycéen peut construire un vrai produit en maîtrisant cet outil. Tu peux lire le code, voir comment il a été construit, et apprendre par l'exemple.

Exemples de requêtes IA utiles quand tu travailles sur ce projet :

Explique-moi pourquoi cette fonction retourne undefined
Propose une version plus lisible de ce bloc CSS
Qu'est-ce qui se passe si l'utilisateur n'a pas de localStorage ?
Revois cette logique de tri et dis-moi si elle est correcte
Comment écrire un bon message de commit pour ce changement ?
Pourquoi ça compte vraiment

L'open source, c'est l'accès au monde tech mondial.

Linux fait tourner 96 % des serveurs mondiaux. Android — 3,6 milliards d'appareils — est open source. React a été ouvert par Meta. VS Code par Microsoft. Kubernetes par Google. Git, l'outil qui gère le code de toutes ces entreprises, a été créé par Linus Torvalds à 21 ans dans sa chambre d'étudiant.

Les États-Unis ont construit la Silicon Valley sur une culture : tu vaux ce que tu as construit, pas où tu as étudié. Un profil GitHub avec de vraies contributions ouvre plus de portes dans une startup américaine qu'un diplôme d'une école que personne ne connaît hors de France. C'est la méritocratie la plus internationale qui existe.

Infrastructure mondiale
Linux, Git, React, Node.js, Python, PostgreSQL — le web mondial tourne sur de l'open source. Pas sur du code propriétaire.
La culture Y Combinator
Les meilleurs incubateurs américains recrutent des hackers qui ont construit quelque chose de réel. Un PR fusionné sur un projet open source est une preuve concrète.
100M+ développeurs
GitHub en 2024 : 100 millions de développeurs actifs. La plus grande bibliothèque de code publique de l'histoire humaine. Accessible depuis le lycée.

« Talk is cheap. Show me the code. »

Linus Torvalds — créateur de Linux et de Git, les deux infrastructures sur lesquelles tourne l'économie numérique mondiale.
